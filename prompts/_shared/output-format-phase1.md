# Phase 1: 独立评审 — 提案输出格式

> 本模板定义所有专家在 Phase 1 独立评审阶段的输出结构。
> 各角色的 ID 前缀、Details 描述和特殊字段由角色 prompt 指定。

## 标准提案格式

提供恰好 **3 个提案**，每个包含：

- **ID**: `{角色前缀}-NNN`（如 MC-001, ED-002）
- **Title**: 一行描述
- **Priority**: P0（必须解决）/ P1（应该解决）/ P2（建议解决）
- **Details**: 3-5 句话，说明问题是什么、为什么重要、如何解决
- **Action**: 具体下一步（如 "增加稳健性检验 X"、"重新定义样本为 Y"）

**至少 2/3 的提案必须与本次会议议题直接相关。**

## Devil Reviewer 特殊格式

Devil Reviewer 使用 **Severity** 替代 Priority：
- **Severity**: FATAL（论文无法继续）/ MAJOR（必须解决）/ MINOR（建议解决）
- 额外字段 **Fix**: 如何修复（若可修复），或为何不可修复
- 额外字段 **VETO**: 若缺陷真正致命且不可修复，使用 VETO 标记

## 格式规范

- 只输出结构化 JSON，不要输出任何摘要或说明
- 使用英文字段名和中英混合内容
