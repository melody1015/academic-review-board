# Academic Review Board — Example Outputs

This directory contains sanitized examples of real review sessions.

---

## Two Types of Examples

| File | Type | What It Shows |
|------|------|---------------|
| [normal-review.md](normal-review.md) | Constructive feedback | Paper is sound, 8 improvements, no fatal flaws |
| [fatal-flaw-detected.md](fatal-flaw-detected.md) | Fatal flaw caught | 7/7 experts caught tautological predictor before submission |

---

## When to Use Which

**Show `normal-review.md` when:**
- Explaining what a typical review looks like
- Demonstrating actionable, constructive feedback
- Showing P0/P1/P2 prioritization

**Show `fatal-flaw-detected.md` when:**
- Telling the "this system saved me" story
- Demonstrating multi-expert convergence
- Showing Devil Reviewer power

---

## Key Takeaways

### Normal Review
- Most reviews are like this — paper is good, needs polish
- P0/P1/P2 prioritization helps focus
- No drama, just improvement

### Fatal Flaw Detection
- Unanimous convergence (7/7) on same issue
- Paper survived after redesign (effect dropped 80% but still significant)
- Self-correction before submission

---

## Real Impact

> "All experts unanimously identified a tautological flaw in my variable definition that I missed entirely."
> — Author, after Session 002

The system finds flaws to **fix**, not reasons to **kill**.
